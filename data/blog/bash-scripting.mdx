### ¿Qué es Bash Scripting?

Bash scripting es el arte de escribir secuencias de comandos en el lenguaje de la shell Bash (Bourne-Again SHell) para automatizar tareas en un sistema operativo tipo Unix. Imagina que tienes una serie de comandos que ejecutas repetidamente. En lugar de escribirlos uno por uno cada vez, puedes guardarlos en un archivo de texto (el script de Bash) y luego ejecutar ese archivo. Bash se encarga de leer las líneas del archivo y ejecutar los comandos secuencialmente.

### Importancia del Bash Scripting:

- **Automatización de tareas:** La principal ventaja es la capacidad de automatizar tareas repetitivas, ahorrando tiempo y reduciendo la posibilidad de errores humanos. Esto puede incluir copias de seguridad, gestión de archivos, despliegues de aplicaciones, monitoreo del sistema, etc.

- **Administración del sistema:** Los administradores de sistemas utilizan extensivamente Bash scripting para gestionar servidores, automatizar tareas de mantenimiento, crear usuarios, gestionar procesos y mucho más.

- **Desarrollo de software:** Los desarrolladores pueden usar scripts de Bash para automatizar tareas de construcción, pruebas, despliegue de aplicaciones y otras tareas relacionadas con el ciclo de vida del software.

- **Personalización del entorno:** Puedes personalizar tu entorno de trabajo creando scripts que realicen tareas específicas que se adapten a tus necesidades.

- **Integración de herramientas:** Bash actúa como un "pegamento" que te permite combinar diferentes herramientas y comandos de Linux para realizar tareas complejas.

### Elementos Fundamentales del Bash Scripting:

- **Comandos:** Un script de Bash está compuesto principalmente por comandos que normalmente ejecutarías directamente en la línea de comandos.

- **Variables:** Puedes almacenar valores en variables para usarlos posteriormente en el script.

- **Estructuras de control de flujo:** Permiten tomar decisiones y repetir bloques de código. Las más comunes son:
  - if, elif, else: Ejecutan diferentes bloques de código basados en condiciones.
  - for: Repiten un bloque de código un número específico de veces o para cada elemento en una lista.
  - while: Repiten un bloque de código mientras una condición sea verdadera.
  - until: Repiten un bloque de código hasta que una condición sea verdadera.
  - case: Ejecuta diferentes bloques de código basados en el valor de una variable.

- **Funciones:** Permiten definir bloques de código reutilizables.

- **Entrada/Salida:** Los scripts pueden recibir entrada del usuario o de otros programas y pueden mostrar salida en la terminal o redirigirla a archivos.

- **Comentarios:** Las líneas que comienzan con # son comentarios y son ignoradas por el intérprete de Bash. Se utilizan para documentar el script y hacerlo más legible.

Vamos a crear algunos scripts sencillos para que te familiarices con los conceptos básicos.

1. Crear el primer script

Abre un editor de texto (como nano, vim, gedit, etc.).

Escribe el siguiente código:

```bash
#!/bin/bash
# Este es mi primer script de Bash
echo "¡Hola, mundo desde Bash!"
```

Guarda el archivo con un nombre descriptivo, por ejemplo, hola.sh. Es una convención usar la extensión .sh para los scripts de Bash.

2. Hacer el script ejecutable

Antes de poder ejecutar el script, necesitas darle permisos de ejecución. Abre tu terminal y navega hasta el directorio donde guardaste hola.sh. Luego ejecuta el siguiente comando:

```bash
sudo chmod +x hola.sh
```

El comando chmod +x añade permiso de ejecución al archivo.

3. Ejecutar el script

Ahora puedes ejecutar tu script de dos maneras:

Ejecutándolo directamente:

```bash
./hola.sh
```

El ./ indica que el ejecutable se encuentra en el directorio actual.

4. Usando variables

Vamos a modificar el script para usar una variable:

Abre hola.sh en tu editor de texto y cambia el contenido a:

```bash
#!/bin/bash
nombre="TuNombre"
echo "¡Hola, $nombre desde Bash!"
```

Guarda el archivo y ejecútalo de nuevo (./hola.sh). Deberías ver:

¡Hola, TuNombre desde Bash!

Nota cómo se accede al valor de la variable usando $nombre. También puedes usar ${nombre}.

5. Obteniendo entrada del usuario

Vamos a crear un script que pregunte por tu nombre:

Crea un nuevo archivo llamado saludo.sh y escribe:

```bash
#!/bin/bash
echo "Por favor, introduce tu nombre:"
read nombre
echo "¡Saludos, $nombre!"
```

Hazlo ejecutable (chmod +x saludo.sh) y ejecútalo (./saludo.sh). El script te pedirá tu nombre y luego te saludará.

6. Estructura de control if

Vamos a crear un script que verifique si un número es positivo:

Crea un archivo llamado positivo.sh:

```bash
#!/bin/bash
echo "Introduce un número:"
read numero

if [ "$numero" -gt 0 ]; then
    echo "$numero es positivo."
elif [ "$numero" -eq 0 ]; then
    echo "$numero es cero."
else
    echo "$numero es negativo."
fi
```

Hazlo ejecutable (chmod +x positivo.sh) y pruébalo con diferentes números.

- [ y ] se utilizan para las pruebas condicionales. Deben estar separados por espacios del contenido de la condición.

- -gt (greater than), -lt (less than), -eq (equal), -ne (not equal), -ge (greater than or equal), --le (less than or equal) son algunos operadores de comparación numérica.

- then indica el bloque de código a ejecutar si la condición es verdadera.

- elif es una abreviatura de "else if".

- else indica el bloque de código a ejecutar si ninguna de las condiciones anteriores es verdadera.

- fi marca el final de la estructura if.

7. Estructura de control for

Vamos a crear un script que itere a través de una lista de frutas:

Crea un archivo llamado frutas.sh:

```bash
#!/bin/bash
frutas="manzana banana naranja fresa"

for fruta in $frutas; do
    echo "Me gusta la $fruta."
done
```
Hazlo ejecutable (chmod +x frutas.sh) y ejecútalo.

- El bucle for itera a través de cada palabra en la variable frutas.

- do marca el inicio del bloque de código a repetir.

- done marca el final del bucle for.

8. Estructura de control while

Vamos a crear un script que cuente hasta 5:

Crea un archivo llamado contador.sh:

```bash
#!/bin/bash
contador=1

while [ "$contador" -le 5 ]; do
    echo "El contador es: $contador"
    contador=$((contador + 1))
done

echo "¡Fin de la cuenta!"
```

Hazlo ejecutable (chmod +x contador.sh) y ejecútalo.

- El bucle while se ejecuta mientras la condición ($contador sea menor o igual a 5) sea verdadera.

- $((...)) se utiliza para realizar operaciones aritméticas.

9. Funciones

Vamos a crear un script que defina una función para saludar:

Crea un archivo llamado funciones.sh:

```bash
#!/bin/bash

saludar() {
    echo "¡Hola, $1!"
}

nombre="Carlos"
saludar "$nombre"
saludar "Ana"
saludar "Luis"
```
Hazlo ejecutable (chmod +x funciones.sh) y ejecútalo.

- Las funciones se definen usando la sintaxis nombre_de_la_funcion() { ... }.

- $1 dentro de la función representa el primer argumento pasado a la función.

- Puedes llamar a la función con diferentes argumentos.

- Las funciones son útiles para organizar el código y evitar la repetición.

Próximos Pasos:

Este es solo el comienzo. Hay muchos más conceptos y herramientas en Bash scripting que puedes explorar, como:

- **Arrays:** Para almacenar listas de valores.
- **Manipulación de cadenas:** Para trabajar con texto.
- **Redirección de entrada/salida:** Para dirigir la entrada y salida de los comandos.
- **Tuberías (|):** Para encadenar la salida de un comando a la entrada de otro.
- **Comandos útiles:** grep, sed, awk, find, cut, sort, etc.
- **Variables de entorno:** Variables que contienen información sobre el sistema y el usuario.
- **Manejo de errores:** Para hacer tus scripts más robustos.

Recursos para aprender más:

- **La documentación oficial de Bash:** Puede ser un poco técnica al principio, pero es la fuente definitiva.

- **Tutoriales en línea:** Hay innumerables tutoriales y cursos gratuitos y de pago disponibles en la web.

- **Libros sobre Bash scripting:** Pueden proporcionar una comprensión más profunda y estructurada.

- **La práctica:** La mejor manera de aprender es escribiendo tus propios scripts para automatizar tareas que realizas regularmente.
